# База 1С для теста шлюза обмена

В папке `1c/` — код обработки **«ТестШлюзаОбмена»** для проверки и отправки данных на HTTP-шлюз проекта (`http://127.0.0.1:8080/api/v1`).

## Что умеет обработка

| Действие | API шлюза |
|----------|-----------|
| **Проверить шлюз** | `GET /api/v1/health` |
| **Отправить на шлюз** | `POST /api/v1/send` (multipart: `file`, `format`, `transform_to`) |
| **Список принятых** | `GET /api/v1/received` |
| **Выбрать файл** | диалог выбора файла на клиенте |

Перед работой в 1С запустите шлюз Go и (при преобразовании формата) Python — см. [`docs/ЗАПУСК_И_ПРОВЕРКА.md`](../docs/ЗАПУСК_И_ПРОВЕРКА.md).

## Вариант А. Внешняя обработка (быстрее всего, ~10 минут)

Подходит, если не нужна отдельная конфигурация — только тест из любой базы.

1. Запустите **1С:Предприятие** → создайте или откройте любую информационную базу (файловую или серверную).
2. **Файл → Новый → Внешняя обработка** (или откройте конфигуратор → внешние обработки).
3. Имя: `ТестШлюзаОбмена`, синоним: `Тест шлюза обмена`.
4. Добавьте реквизиты обработки по таблице из [`ТестШлюзаОбмена/РеквизитыФормы.txt`](ТестШлюзаОбмена/РеквизитыФормы.txt) (в т.ч. **АдресДанныхФайла** — для тонкого клиента).
5. **Модуль объекта** — скопируйте целиком из [`ТестШлюзаОбмена/ObjectModule.bsl`](ТестШлюзаОбмена/ObjectModule.bsl).
6. Создайте **основную управляемую форму**, добавьте команды и элементы (см. тот же файл с реквизитами).
7. **Модуль формы** — скопируйте из [`ТестШлюзаОбмена/FormModule.bsl`](ТестШлюзаОбмена/FormModule.bsl).
8. Сохраните как `ТестШлюзаОбмена.epf` (**Файл → Сохранить как**).
9. В режиме **1С:Предприятие**: **Файл → Открыть** → выберите `.epf`.

### Проверка

1. Убедитесь, что шлюз отвечает: `http://127.0.0.1:8080/api/v1/health`.
2. В обработке: **Проверить шлюз** → сообщение вида `Шлюз доступен: {"status":"ok","version":"v1"}`.
3. **Выбрать файл** → укажите `examples/sample.xml` из корня курсового проекта (полный путь на диске).
4. **Формат:** `xml`, **Преобразовать в:** `json` (если Python запущен).
5. **Отправить на шлюз** → код **201**, в ответе `id` записи.
6. **Список принятых** — JSON-массив с новой записью.

## Вариант Б. Обработка внутри своей конфигурации (отдельная «база»)

Если нужна именно **своя конфигурация** для курсовой:

1. Конфигуратор → **Конфигурация → Создать конфигурацию** (или новая ИБ с пустой конфигурацией).
2. Имя конфигурации, например: `ШлюзОбмена`, режим: **Управляемое приложение**.
3. В дереве: **Обработки** → Добавить → `ТестШлюзаОбмена`.
4. Повторите шаги 4–7 из варианта А (реквизиты, модули, форма).
5. **Конфигурация → Обновить конфигурацию БД**.
6. Запуск **1С:Предприятие** → раздел с обработками → **Тест шлюза обмена**.

Сохраните ИБ в отдельную папку (например `1c/ИБ_ШлюзОбмена/`) для сдачи работы.

## Важно для файлового режима и сервера

- HTTP-запросы выполняются **на сервере 1С** (`&НаСервере`). Путь к файлу (`ПутьКФайлу`) должен быть доступен **с того компьютера, где работает сервер 1С** (для файловой ИБ — обычно ваш ПК).
- Если используете **тонкий клиент** к удалённому серверу — кладите тестовый XML на сервер или используйте общий сетевой путь.
- Для **localhost** файловая база на том же ПК, что и шлюз Go, — всё работает с путём вида `C:\...\Курсовая\examples\sample.xml`.

| Реквизит | По умолчанию |
|----------|----------------|
| АдресСервера | `127.0.0.1` |
| Порт | `8080` |

## Тонкий клиент

Кнопка **«Выбрать файл»** использует `НачатьПомещениеФайла` (не `ДиалогВыбораФайла.Выбрать()` — он недоступен в тонком клиенте). Файл попадает во временное хранилище сервера 1С, отправка идёт через `ОтправитьДвоичныеДанныеНаШлюз`.

Поле **ПутьКФайлу** можно по-прежнему заполнить вручную полным путём на **сервере** 1С (без выбора файла) — тогда используется чтение с диска.

## Типичные ошибки

| Симптом | Причина |
|---------|---------|
| «Конструктор не найден ДиалогВыбораФайла» | Старый модуль формы — обновите из `FormModule.bsl` |
| «Соединение не установлено» | Не запущен `server.exe` / Docker gateway |
| Код 502 при отправке с преобразованием | Не запущен Python на порту 5000 |
| Файл не найден | Путь недоступен с сервера 1С |
| Код 422 | Невалидный XML/JSON в файле |
