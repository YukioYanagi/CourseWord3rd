# Запуск сервера и проверка работоспособности

Проект состоит из двух процессов:

1. **Python** — сервис трансформаций JSON/XML/SOAP (порт **5000** по умолчанию).
2. **Go** — HTTP-шлюз с версией API **v1**, логами и веб-страницей (порт **8080** по умолчанию).

Рабочая папка в командах ниже — корень проекта «Курсовая» (где лежат `go.mod`, папки `cmd`, `python`, `web`).

---

## 1. Подготовка (один раз)

### Python

```powershell
cd python
python -m pip install -r requirements.txt
cd ..
```

### Go

Установите [Go](https://go.dev/dl/) 1.26.3+ (или новее) и выполните в корне проекта:

```powershell
go build -o server.exe ./cmd/server
```

---

## 2. Запуск

### Вариант через Docker (если установлены Docker и Docker Compose)

Из корня проекта:

```powershell
docker compose up --build
```

После старта: веб-интерфейс шлюза — `http://localhost:8080/`, сервис Python доступен на хосте как `http://localhost:5000/health`. Данные шлюза сохраняются в томе `gateway-data`.

---

### Локально (два процесса)

Откройте **два** окна терминала (или две вкладки).

#### Окно A — Python

```powershell
cd python
python -m uvicorn app:app --host 127.0.0.1 --port 5000
```

Должна появиться строка вида: `Uvicorn running on http://127.0.0.1:5000`.

#### Окно B — шлюз Go

Из **корня** проекта:

```powershell
.\server.exe
```

По умолчанию слушает адрес **`:8080`**. Если порт занят (ошибка `bind: Only one usage...`), укажите другой:

```powershell
$env:HTTP_ADDR = ":8090"
.\server.exe
```

Дополнительные переменные окружения (необязательно):

| Переменная | Назначение | По умолчанию |
|------------|------------|----------------|
| `HTTP_ADDR` | Адрес прослушивания Go | `:8080` |
| `PYTHON_TRANSFORM_URL` | URL сервиса Python | `http://127.0.0.1:5000` |
| `DATA_DIR` | Каталог для `received` и `index.json` | `data` |

## 3. Проверка работоспособности

### Вариант А — браузер

1. Откройте `http://127.0.0.1:8080/` (или тот порт, который задали в `HTTP_ADDR`).
2. Вкладка **«Отправить файл»** — выберите файл из папки `examples` (`sample.json`, `sample.xml`, `sample-soap.xml`), укажите соответствующий формат, при необходимости включите преобразование и нажмите **Отправить**.
3. Вкладка **«Принятые файлы»** — должна появиться новая строка; ссылка **скачать** отдаёт сохранённый файл.

### Вариант B — скрипт PowerShell

Из корня проекта (при необходимости задайте адрес шлюза, если не 8080):

```powershell
$env:GATEWAY = "http://127.0.0.1:8090"   # только если шлюз не на 8080
powershell -ExecutionPolicy Bypass -File .\examples\check.ps1
```

Скрипт `examples/check.ps1` (латиница в имени — меньше проблем с кодировкой в PowerShell) проверяет `/health` у Python, `GET /api/v1/health` у Go, выполняет `POST /api/v1/send` с `examples/sample.json` и выводит `GET /api/v1/received`.

### Вариант C — вручную (curl)

Проверка шлюза:

```text
curl.exe -s -D - http://127.0.0.1:8080/api/v1/health
```

В заголовках ответа должна быть версия API (например `X-Api-Version: v1`), в теле — JSON `{"status":"ok","version":"v1"}`.

Отправка файла (путь к `sample.json` подставьте свой):

```text
curl.exe -s -F "file=@examples\sample.json" -F "format=json" http://127.0.0.1:8080/api/v1/send
```

Список принятых:

```text
curl.exe -s http://127.0.0.1:8080/api/v1/received
```

## 5. Логи

Запросы к шлюзу Go пишутся **в консоль** в формате JSON (поля `method`, `path`, `status`, `duration_ms` и т.д.). Для отчёта по курсовой можно приложить скрин консоли или перенаправить вывод в файл:

```powershell
.\server.exe 2>&1 | Tee-Object -FilePath gateway.log
```

## 6. Типичные проблемы

| Симптом | Что сделать |
|---------|-------------|
| `bind ... 8080` занят | Запустите с `$env:HTTP_ADDR=":8090"` и открывайте сайт на этом порту. |
| **WinError 10048** на порту **5000** (одно использование адреса сокета) | Порт занят, чаще всего уже запущен **uvicorn**. **Вариант 1:** найти PID: `netstat -ano` (строка с `127.0.0.1:5000` и `LISTENING`) → `taskkill /PID <номер> /F`. **Вариант 2:** другой порт, например **5001**: `python -m uvicorn app:app --host 127.0.0.1 --port 5001`, перед запуском Go: `$env:PYTHON_TRANSFORM_URL="http://127.0.0.1:5001"`. |
| Ошибка преобразования / 502 при «Сохранить как» | Убедитесь, что Python запущен на `127.0.0.1:5000` или задайте `PYTHON_TRANSFORM_URL`. |
| JSON «не проходит» после сохранения из PowerShell | Файл мог получить BOM; шлюз обрезает UTF-8 BOM при приёме. Лучше сохранять UTF-8 без BOM или использовать `examples/sample.json`. |
